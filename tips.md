# 🔧기본 베이스로 가져가는 Coding Tips

## [접근성 체크해보기]
- [validator.w3.org](http://validator.w3.org) : 접근성에 알맞은 사이트인지 체크해볼 수 있다
- 웹표준으로 문제가 없다는 뜻
- ***포폴 만들고나서 밸리게이션 체크 꼭 해주기!!!***
-   텍스트는 IR기법으로 사라지게 하기
    - 이미지태그 alt값 표시대신 밑에 strong태그 넣어서 설명 가능
    - alt값을 빼버리면 스크린리더가 이미지 경로를 읽음. 그래서 alt=”” 로 빈공간 넣어주기
- section에는 헤딩태그 반드시 써주기.
- 섹션사용할때는 반드시 헤딩넣을만한 내용이 있어야 함.
- 이미지 꾸밈 요소는 alt값 없애기. 스크린리더에 방해됌.
- 이미지를 img태그로 넣을때는 설명이 필요한 이미지, 설명이 필요 없으면 css의 background-image
- h1으로 넣기에는 제목이 좀 부담이면 > h2로 넣자. article로 사용할 경우 다른페이지에서도 단독으로 사용해야 하는데 다른 페이지의 h1과 겹칠수도 있어서.
-   버튼 안에 p태그 넣지 말기   
-   형제요소끼리는 헤딩레벨 맞춰주기
- 닫기버튼을 제일 밑에 만들어두는 이유는 스크린리더 읽었을때 내용을 다 보여주고 닫게하기 위함때문. 추후 position으로 디자인적으로만 위로 올리기

## [모듈화 작업 가이드]
- 요소에 width:100% 쓰는건 어느 컨테이너에 들어가던 잘 맞춰서 들어갈 수 있도록 **모듈화** 시키는 것
- 모듈을 짠 다음에 조립하는 과정을 거치면 서로 분담해서 작업하기에 매우 좋다. 마치 프라모델의 런너 만드는 느낌으로.
- 추후 HTML 코드 또한 모듈식으로 만들면 평가는 매우 좋을 것.
- 크롬은 최소 폰트단위가 10px
- float, flex, grid 섞어서 써도 무방
- 자잘한 요소도 height 남겨두지 말고, 왠만하면 다 높이는 없애두기. 유연한 개발자가 되어야한다.
-   로고에는 h1태그 넣기.
-   section 안에 다시 section 써도 좋다
-   article은 작은 html으로 생각.
    - 광고 관련은 article보다는 aside로 표현 (광고 또한) 
-   위젯같은거는 article로 들어가도 좋음. 어딘가에 똑 떨어져 붙여서 사용해도 문제없는게 article. aside도 비슷.
-   section 쓰면 헤딩태그가 들어가는게 권장. 주제가 있어야하기때문에. h1넣더라도 디자인적으로 넣지 않음. 접근성과 seo를 위하여.
-   address 태그는 회사주소 등의 연락부분에 넣기
    -   address >dl > dt+dd 주소쓰는 방법이 주된 방법. 타이틀, 발매일 가격 또한 마찬가지
-   ul이나 ol안에는 반드시 li만 들어옴.
